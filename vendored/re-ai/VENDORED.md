## What's vendored

- `servers/re-<name>/` — the Python server code (`pyproject.toml`, `src/`, `bin/`). `.git/`, `LICENSE`, and `README.md` stripped.
- `data/anti-analysis-catalog.json` — the anti-analysis correlation catalog.

## What's NOT vendored

- `.venv/`, `uv.lock`, `__pycache__/`, `*.pyc`, `*.egg-info/` — excluded (regenerated by `pip install -e .`).
- The output triage directory (`output/`) — not vendored. RE-BREAKER reads triage data at runtime, not from vendored files.

## How this differs from `.mcp.json`

The 8 vendored servers are **library code**, not MCP tools. They are NOT registered in `.mcp.json` and will NOT be picked up by the MCP harness. RE-BREAKER's own servers (`servers/re-catalog-match/`, `servers/re-runtime-dump/`, etc.) call into the vendored code directly via Python imports, using the vendored path as the import root.

The `RE_AI_PLUGIN_ROOT` env var is NOT read by any non-vendored RE-BREAKER code. The vendored code is self-contained.

## Sync policy

To update the vendored servers to a newer public release:

```bash
# For each server:
cd vendored/re-ai/servers/re-<name>
git init && git remote add origin https://github.com/Heretek-RE/re-<name>.git
git fetch --depth 1 origin v0.2.0  # or whatever tag
git checkout FETCH_HEAD
rm -rf .git LICENSE README.md
```

Or use `git subtree`:

```bash
git subtree add --prefix=vendored/re-ai/servers/re-<name> \
  https://github.com/Heretek-RE/re-<name>.git v0.2.0 --squash
```

After updating, re-run the stress tests and diff against the baseline.
